# Website Next Steps Todo Lists

Copy this into Google Docs and use it as the shared next-steps tracker for the project.

## Client Todo List

- Confirm the final business name, phone number, email address, and service area wording
- Confirm business hours and lead response-time wording
- Confirm whether claims like licensed, insured, bonded, years in business, and projects completed are accurate
- Fill out the Website Content Workbook for the homepage
- Fill out the About page story, values, and proof points
- Fill out the Interior, Exterior, and Cabinet service page content
- Provide 10-20 approved gallery photos with file names and short descriptions
- Provide 5-8 final FAQ questions and answers
- Decide whether testimonials should be added to the live site
- Provide 3-5 testimonials if they want that section included
- Review the site in laptop view and report bugs using the Laptop Bug Report Guide
- Review the site in mobile view and report bugs using the Mobile Bug Report Guide
- Approve final CTA wording such as "Get Quote" and "Get Free Estimate"
- Approve final image choices for hero sections and supporting page images
- Confirm any required legal wording or disclaimers before launch

## Developer Todo List

- Review the completed client workbook and replace placeholder copy across all pages
- Replace placeholder contact details and business claims with verified information
- Update global content so the header, contact block, and footer stay consistent across the site
- Expand the gallery page beyond the current 3 live images once real project photos are approved
- Add alt text and cleaner organization for every final gallery image
- Decide whether to bring the testimonial section from the backup file into the live site
- Test the contact form end-to-end with a real submission and confirm delivery to the client
- Run a full responsive QA pass at desktop, tablet, and mobile sizes
- Check the Services dropdown behavior on hover and tap after final content changes
- Fix or replace the footer Gallery links on internal pages because they currently point to `index.html#gallery`, but the homepage does not have a `#gallery` section
- Decide whether the homepage should gain a gallery preview section or whether footer links should point to `gallery.html` instead
- Consider adding a heading or intro text to the Gallery page to improve clarity and make bug reporting easier
- Review whether testimonials, before/after examples, or stronger trust signals should be added before launch
- Final proofread for spelling, city names, link accuracy, and contact consistency

## Suggested Work Order

1. Client completes the content workbook
2. Client sends images, FAQs, and approvals
3. Developer updates the website with final content
4. Client reports laptop and mobile bugs
5. Developer fixes bugs and does final QA
6. Client gives final approval to launch
